PE-Sleuth

融合程序级语义与静态特征，借助大语言模型实现可解释的勒索软件检测

程序级语义 静态特征融合 大语言模型 可解释 AI 零样本 + 微调

1. 背景与动机

勒索软件（Ransomware）以加密文件并索要赎金为核心攻击手段，已成为企业和公共机构面临的 最严重安全威胁之一。尤其在 Windows 平台上，可移植可执行文件（PE）是软件分发的主要载体， 基于 PE 文件的静态检测因此具有重要的工程价值。

传统方法的三大瓶颈

特征覆盖有限、对抗鲁棒性不足

缺乏可解释的审计证据

跨领域自适应的重训练成本高

PE-Sleuth 的应对策略

程序级语义 + 静态特征双源融合

LLM 生成人类可读的推理依据

零样本基座 + LoRA 微调双模式

现有方法大致分为三类：签名/规则方法轻量但易被混淆绕过；手工特征 + 机器学习 依赖 PE 头、字符串等有限特征，泛化能力弱；深度端到端模型闭集准确率高但缺乏可审计证据， 且在分布偏移下泛化较差。此外，目前将 LLM 用于 PE 勒索软件检测的研究极为稀缺——代表性工作 LAMD 仅针对 Android 应用，聚焦于安全敏感 API 的局部上下文，难以应对 Windows/PE 场景下沿调用流和数据流 分布的恶意逻辑。

核心洞察：PE-Sleuth 的出发点是——恶意代码的语义意图往往超越原始字节模式， 深藏于控制流、数据引用和系统调用之中。通过反编译 + 调用图划分 + 层次化语义抽象， 可以从二进制代码中提取出 LLM 可理解、可推理的程序级语义表示。

3. 各阶段详细设计

3.1 阶段一：反编译（Decompile）

🔧 二进制 → C 源代码

使用 IDA Pro 的 idat.exe 进行批量自动化反编译，调用 Hex-Rays 反编译器将每条 x86/x64 指令翻译为结构化的 C 代码。这一步骤是连接"原始二进制"与"LLM 可读表示"的关键桥梁—— 低级指令被转换为暴露控制流结构、数据引用和语义线索的 C 源码， 使得后续的 LLM 分析成为可能。

反编译参数 -Ohexrays:new:-nosave:target name:ALL 确保所有函数均被完整导出， 生成 .c 文件供下游阶段使用。

3.2 阶段二：语义抽象（Semantic Abstraction）

这是 PE-Sleuth 的核心创新环节。由于 LLM 的上下文长度有限（本文设定为 32,768 token）， 无法一次性摄入完整的反编译源码。语义抽象阶段通过三个协同子步骤，在保证语义完整性的前提下 将长代码转换为紧凑的程序级语义：

📐 子步骤 1：FCG 划分（FCG Partitioning）

从反编译 C 代码中解析函数列表和调用边，构建函数调用图（Function Call Graph, FCG）。 将 FCG 划分为连通子图，按照每块 32,768 token 的严格上限打包。关键策略包括：

  • 调用邻近优先：调用关系紧密的函数优先归入同一块，保持语义连续性
  • 语法感知切割：超大函数优先在花括号边界处分割，避免破坏语法结构
  • 滑动窗口兜底：若仍超出限制，使用滑动窗口确保每个片段可处理
  • 哈希去重：消除跨块的重复函数体，减少冗余分析
  • 贪婪合并：初始划分后，将相邻小块合并至 98% 容量上限，减少 LLM 调用次数

可视化与质量控制：FCG 和块分配以彩色图形式渲染（如原论文 Fig. 2 所示）—— 节点表示函数，边表示调用关系，颜色标识块归属。不连通子图自然映射到不同块， 过大子图按调用邻近性分割。

🔬 子步骤 2：块级分析（Block-level Analysis）

每个代码块被 LLM 以简洁、中立的提示进行分析。LLM 被约束为"代码行为提取器" 的角色，输出该块的块级语义描述。关键设计决策：

  • 刻意避免良性/恶意暗示：提示中不包含任何分类倾向，降低偏差
  • 保留非判别性行为：即使某些行为不与勒索软件直接相关，仍然记录——这些信息对后续推理有价值
  • 行为而非标签：输出聚焦于"这段代码做了什么"而非"它看起来可疑吗"

🧩 子步骤 3：程序级分析（Program-level Analysis）

所有块级语义被合并提交给 LLM，提炼为统一的程序级语义。提示要求 LLM：

  • 给出紧凑的整体评估
  • 突出最具信息量的行为及其理由
  • 为下游分类呈现高价值的语义信号

这种"先分后合"的层次化抽象策略，既克服了 LLM 的上下文长度限制， 又确保了完整程序的语义意图得以保留——这是 PE-Sleuth 相比仅关注局部 API 上下文的方法 的核心优势所在。

3.3 阶段三：静态特征提取（Static Features Extraction）

📋 双源融合的第一极——全局静态线索

在严格 1,000 token 的长度预算下，从反编译源码和 IDA 注释中提取并归一化静态特征， 作为程序级语义的互补信号。该阶段同样包含两个子步骤：

提取（Extraction）—— 四类特征

📝

字符串

代码中的硬编码字符串，常包含域名、路径、注册表键、勒索信息等关键线索

📥

外部 API 导入

程序依赖的系统 API，即使导入表被混淆，反编译仍可揭示实际调用的 API

🌐

全局声明

全局变量声明，揭示程序的状态管理和数据流转模式

⚠️

可疑指标

卷影副本删除、大规模加密、勒索信逻辑、密码学初始化等典型行为信号

归一化（Normalization）—— 预算感知裁剪

  • 规范化 & 去重：统一格式，消除重复项
  • 预算感知排序：按重要性排列，每类配额约 60 字符串 / 40 导入 / 30 声明
  • 确定性裁剪：优先移除低优先级条目，确保高风险匹配和主调用路径上的条目得以保留
设计意图：静态特征虽然来自"表层"信息，但描绘了程序的能力边界—— 文件/注册表/服务控制、进程终止、密码学操作——即使函数级语义分析不够深入时， 静态特征仍能提供关键的全局佐证。1,000 token 的预算约束确保该信息源保持轻量。

3.4 阶段四：分类与推理依据（Classification & Rationale）

🏷️ 子步骤 1：分类（Classification）

将程序级语义 + 归一化静态特征融合为中间表示（IR）， 提交给 LLM 进行二分类判决。分类策略包含两层机制：

强信号规则（降低漏报）

在 LLM 推理之前，先进行规则匹配。若 IR 中包含以下任一强信号， 样本直接标记为 RANSOMWARE：

🔴 卷影副本删除

vssadmin、delete shadows 等命令/字符串

🔴 勒索通信

.onion 地址、勒索相关短语

🔴 解密行为触发

包含解密相关逻辑（暗示加密已执行）

🔴 密码学 API 调用

CryptAcquireContext 等初始化 API

🔴 大规模文件加密

程序级语义指示批量加密行为

🔴 服务终止

为便于加密而终止安全/备份服务

LLM 整体判决（无强信号命中时）

若无强信号命中，则由 LLM 基于 IR 进行整体性综合判定。 文中评估了两种 LLM 配置：

  • 零样本基座模型（Base）：直接使用 Qwen3-14B 的预训练能力进行推理
  • LoRA 微调模型（Fine-tuned）：在训练集上以秩 24、alpha 48、dropout 0.15 进行监督微调， 使用 4-bit 量化降低显存占用

📝 子步骤 2：推理依据生成（Rationale Generation）

分类结果与 IR 再次提交给 LLM，生成面向安全分析师的简明推理依据。 提示聚焦于三个维度：

  • 触发证据：哪些具体的行为/特征导致了该分类决策
  • 受影响目标：哪些文件、服务或系统组件是攻击的目标
  • 潜在误报分析：是否存在良性软件可能表现出类似行为的场景

这一步骤是 PE-Sleuth "可解释性"承诺的关键实现——它将分类决策与 数据流路径、API 调用链等具体证据链接起来，支持人工审计和结果复现。

🔍 零样本场景

仅程序级语义（F1=0.7870）接近全量（F1=0.8077），远优于仅静态特征（F1=0.4895）。 表明无监督信号时，程序级语义更能被模型有效利用。

🎯 有监督场景

仅静态特征（F1=0.9470）接近全量（F1=0.9703），远超仅程序级语义（F1=0.7948）。 表明固定结构的静态特征更有利于参数更新学习稳定判别特征。

🤝 融合增益

无论基座还是微调，全量输入均优于任一单源输入，验证了程序级语义与静态特征的互补性。

📊 与基线对比

PE-Sleuth 性能优于或持平 MLP、CNN、PCA-NB 三种代表性方法，同时额外提供可解释输出。

5. 设计亮点总结

🏗️

层次化语义抽象

FCG 划分 → 块级分析 → 程序级聚合。先分后合，既克服 LLM 上下文长度限制， 又保留完整程序的语义意图，对加壳和间接调用具有更强鲁棒性。

🔗

双源信息融合

深度语义（程序级语义）与广度线索（静态特征）互补—— 前者捕获行为意图，后者描绘能力边界。实验结果证明 1+1 > 2。

🧠

LLM 驱动的推理

利用 LLM 的代码理解与推理能力进行二分类，突破传统手工特征和端到端模型在可解释性上的局限。

🎯

强信号规则兜底

在 LLM 推理前进行规则匹配，对高风险模式零容忍，有效降低漏报率。

🔧

LoRA 高效微调

以极低参数开销（rank=24）实现显著性能提升，支持在有限算力下的领域自适应， 已开源微调权重。

📝

可审计的推理依据

分类决策附带触发证据、受影响目标和误报分析，将黑盒预测转化为透明、 可复现的分析报告。

🔬

消融实验设计

系统性评估了不同信息源（语义 vs 静态）和不同训练范式（零样本 vs 微调）的贡献， 揭示了它们在零样本/有监督场景下的差异化优势。

🌐

全本地化部署

整个工作流本地化运行，不调用外部 API，使用 4-bit 量化在单卡 48GB GPU 上 运行 14B 参数模型，具备工程可行性。

6. 适用场景与部署建议

🎯 部署策略

🏢 企业安全运营

推荐使用 Fine-tuned-Full 配置（双源 + 微调），实现高召回与高精确率的最佳平衡， 附带推理依据支持安全分析师快速研判。

🔬 威胁狩猎（零样本）

在缺乏标注样本的新领域/新家族检测中，优先使用程序级语义（Base-Semantics）， 无需重训练即可获得接近全量的性能。

⚡ 高吞吐场景

若计算资源受限或需极高吞吐量，可考虑 Fine-tuned-Static（仅静态特征）—— 跳过语义抽象阶段，以小幅性能损失换取显著加速。
